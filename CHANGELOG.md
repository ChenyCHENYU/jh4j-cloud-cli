# Changelog

## 0.6.1

- 增加统一 JH4J 终端主题，真彩终端使用青色到靛蓝的品牌渐变，并兼容 `NO_COLOR` 与非 TTY 环境。
- 按语义区分交互色彩：青色表示重点、绿色表示成功、黄色表示待办、灰色表示辅助信息。
- 重构完成态信息层级，将项目摘要、本地服务、工程配置和运行状态分区展示。
- 将下一步命令独立为紧凑的主题色面板，移除大面积绿色结果边框。
- 模板拉取与项目生成使用统一品牌色加载动画。

## 0.6.0

- 直接展示 PC 管理端与移动端 H5 模板，并为每个选项提供独立技术栈提示。
- 增加“快速创建”和“自定义创建”两种方式；快速创建直接采用模板推荐值，自定义创建才询问项目名、标题、端口和联调地址。
- 下载与初始化过程改为轻量加载状态，不再显示冗余计时标签。
- 新增创建完成结果面板，集中展示模板版本、实际配置、预设能力、依赖与 Git 状态以及下一步命令。
- 用户取消时正常收尾，生成异常使用统一终端样式展示。

## 0.5.1

- 默认交互只选择项目类型并确认项目名；项目名直接回车使用默认值，也可以现场修改。
- 标题、端口、本地服务地址和其他项目参数静默使用模板默认值。
- 精简创建完成输出，只保留项目结果、后续命令和配置文件位置。

## 0.5.0

- 精简交互流程：单模板自动选择，标准能力直接采用模板默认值。
- 模块标识自动从项目名推导，不再询问 Registry、public 地址和五套环境配置。
- 默认不安装依赖，创建完成后提示开发者手动执行 `pnpm install`。
- 增加显式 `--install` 参数，仍可按需启用自动安装。
- 完成提示中列出项目配置、Registry 和环境文件的修改位置。
- 只展示当前可用的项目类型，并使用互不混淆的固定技术栈标签。
- 项目名自动转为小写短横线格式，标题、端口和 URL 在输入时就地校验。
- 模板下载阶段显示进度状态，单模板自动选择后明确显示模板名称。

## 0.4.0

- 接入 `mobile.robot-h5` 移动端 H5 模板，支持 GitHub/Gitee 双 HTTPS 源。
- 移动端创建流程使用应用标题、本地 API 地址等移动端语义，隐藏 PC 专属问题。
- CLI 与直接 clone 共用移动端模板初始化契约。
- 移除 `--ref` 的硬编码 `main` 默认值，允许各模板使用 Catalog 中声明的分支或标签。
- 重写中文 README，使说明与当前命令和能力保持一致。

## 0.3.0

- 增加“项目类型 → 具体模板 → 标准能力”的分层交互。
- 后端与移动端分类在无模板时明确显示为暂未提供。
- 增加通用模板能力协议及 `--features`、`--no-standards` 参数。
- PC 模板接入完整 `@robot-admin/git-standards` 可选能力。
- 调整 Git 初始化顺序，确保依赖安装时 Husky 能正确激活。
- 发布版默认使用 GitHub/Gitee HTTPS 主备源拉取模板。
- npm 包迁移到 `@agile-team/jh4j-cloud-cli` 并配置为公开发布。

## 0.2.0

- 增加外部模板 Catalog 合并与覆盖。
- 增加用户级配置文件和 `config` 命令。
- 增加 Git、HTTP/local tar 模板源与缓存管理。
- 创建过程改为 staging 事务和失败回滚。
- 增加 `--config`、`--no-cache` 和 JSON 输出。
- 增加缓存、Catalog、配置文件与事务故障测试。

## 0.1.0

- 首次实现 `create`、`list`、`doctor`、`info` 和模板校验。
- 接入 `jh4j-ui-template@1.0.0`。
